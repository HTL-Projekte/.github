## 🚀 Mitmachen

Du möchtest deine eigenen Projekte teilen oder die Listen ergänzen?

### 🔐 Voraussetzung für eine Mitgliedschaft

Um Mitglied der Organisation zu werden, musst du mindestens den **Advanced-Stand** aus unserem
👉 [Git-Intro](https://github.com/HTL-Projekte/.github/wiki/Git-Intro) beherrschen.

Das bedeutet insbesondere:

* Sicherer Umgang mit Branches
* Arbeiten mit Pull Requests
* Verständnis von Merge-Konflikten
* **Forking-Workflow (sehr wichtig!)**
* Saubere Commit-Messages
* Nutzung von Issues

Die Organisation arbeitet mit einem strukturierten Workflow. Direkte Commits auf `main` sind nicht vorgesehen.

---

## 🎓 Warum ist „Advanced“ Voraussetzung?

Die Organisation **HTL-Projekte** arbeitet mit einem strukturierten, professionellen Git-Workflow.

Eine Mitgliedschaft setzt daher mindestens den **Advanced-Stand im Git-Intro** voraus, um:

* sauberes Arbeiten mit Branches und Pull Requests sicherzustellen
* Merge-Konflikte eigenständig lösen zu können
* den `main`-Branch stabil zu halten
* und Projekte langfristig nachvollziehbar zu dokumentieren

Besonders wichtig ist der sichere Umgang mit dem **Forking-Workflow**, da dieser kontrollierte Mitarbeit ohne Risiko für das Hauptprojekt ermöglicht.

Die Voraussetzung dient der Qualitätssicherung und der Vorbereitung auf reale Industrie-Standards.

---

### 📌 Schritte zur Aufnahme

1. **Vorbereitung:** Git-Intro bis mindestens „Advanced“ durcharbeiten.
2. **Beitreten:** Erstelle ein [Issue](https://github.com/HTL-Projekte/.github/issues), um eine Einladung zur Organisation zu erhalten.
3. **Dokumentieren:** Lies die [Abgabe-Regeln](https://github.com/HTL-Projekte/.github/wiki/Regeln-Abgaben).
4. **Eintragen:** Verlinke dein Repository im Wiki auf der entsprechenden Fach-Seite.
